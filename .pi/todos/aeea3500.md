The web-search skill works with Anthropic models but not DeepSeek. Investigate and fix.

## Fix Complete

### Root Cause
The `search.mjs` script only supported 3 providers (anthropic, openai-codex, openrouter) for native web search. When `defaultProvider` was set to `deepseek`, `normalizeProvider()` returned undefined, causing it to fall through to the openrouter auth key which was rate-limited.

### Fix
1. **Added DuckDuckGo Lite fallback**: Free, no API key needed, always works. Uses the `https://lite.duckduckgo.com/lite/` endpoint which is simple and reliable.
2. **Updated `normalizeProvider`**: Now recognizes `deepseek`, `mistral`, and `duckduckgo` as valid providers.
3. **Updated `pickProvider`**: Auto-detects the best provider:
   - Native search (anthropic/openrouter/codex) if credentials available
   - Falls back to `duckduckgo` for deepseek/mistral (no native web search)
   - Always has DuckDuckGo as last resort
4. **Updated SKILL.md**: Clear documentation about how it works with any provider.

### Tested
- `node search.mjs "capital of France" --purpose "test"` → returns real search results via DuckDuckGo
